# Lista 07 — Missão: Do Rascunho ao Protótipo

!!! abstract "Briefing da Missão"
    Projete antes de codar ([Aula 07](../aulas/07-aula.md)): crie **wireframes**, um protótipo navegável e um plano de teste de usabilidade.

## 🎯 Exercícios

??? abstract "Missão 1 — Wireframe"
    Desenhe (papel ou Figma) o wireframe de **3 telas** (lista, detalhe e formulário) e indique com setas o fluxo entre elas.

??? abstract "Missão 2 — Protótipo navegável"
    No **Figma**, torne o wireframe clicável ligando botões às telas. Inclua o **link compartilhável** do protótipo no README.

??? abstract "Missão 3 — Roteiro de teste"
    Escreva um roteiro de teste de usabilidade: 3 tarefas, o que observar e como medir sucesso. (Bônus: aplique com um colega e relate.)

## 📁 Estrutura sugerida

```text
uni-udwmj-lista07/
├── README.md         # link do Figma + roteiro de teste
└── wireframes/       # imagens dos rascunhos/telas
```

## 🚀 Passo a passo da entrega

1. Crie o repositório **público** **`uni-udwmj-lista07`**.
2. Coloque o link do Figma e o roteiro no `README.md`; exporte os wireframes em `wireframes/`.
3. Suba o conteúdo:
   ```bash
   git init
   git add .
   git commit -m "docs: resolve lista 07"
   git branch -M main
   git remote add origin https://github.com/SEU_USUARIO/uni-udwmj-lista07.git
   git push -u origin main
   ```
4. **Entregue o link do repositório** no **Google Classroom** (garanta que o link do Figma esteja com acesso liberado).

!!! check "Checklist de qualidade"
    - [ ] Fluxo entre telas indicado claramente.
    - [ ] Protótipo do Figma **navegável** e com acesso liberado.
    - [ ] Roteiro com tarefas, critérios de observação e de sucesso.
    - [ ] Repositório público `uni-udwmj-lista07` e link no Classroom.
